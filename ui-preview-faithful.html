Splash & Auth

Cold-boot splash, welcome, two-tab login (Phone / Email), OTP verification, and the full single-page signup form. Every screen keeps its static route factory, its MobX store (LoginStore / SignupStore), and its Firebase / SharedPreferences wiring verbatim — the upgrade is visual only.

Dashboard

Bottom-nav shell (DashboardScreen) hosts 5 tabs via IndexedStack: Home, Subscribe, Progress, Leaderboard, Cortex.ai. Home itself is composed of 4 vertical bands — Hero, Continue, Learn From, Discover. Every MobX store call, route argument, and SharedPreferences flag is preserved.

Profile & Settings

Edit profile (full-form seeded from HomeStore.userDetails), about, delete-history confirmation, theme sheet.

04 · Video Lectures

5-level drill from category → subcategory → topic → chapter → player. Gradient brand header, 28-radius sheet, 6-chip filter row, stats footer (Completed · In Progress · Not Started · Bookmarked · Offline). Faithful reconstructions of video_category.dart, video_subject_detail.dart, video_topic_category.dart, video_chapter_detail.dart, video_player_detail.dart.

05 · Notes (PDF library)

Notes drill from category → subject → chapter → topic → PDF reader. AppTokens-based _HeroHeader + _FilterChipRow · Observer(NotesCategoryStore.notescategory) filtered client-side by completedPdfCount / progressCount / notStart / offlineCounts / bookmarkPdfCount · PSPDFKit on mobile, Syncfusion SfPdfViewer on desktop · hero bar + Save FAB flushes annotations back.

Hard Copy + Hardcopy Notes

Two related flows: hardcopy/ (PDF reader for owned books — PurchaseHardcopyScreen → HardcopyDetailsScreen → VolumeIndexScreen → ChapterDetailsScreen) and hardcopyNotes/ (catalogue → cart → checkout with SubscriptionStore).

06 · Live Classes

DefaultTabController-driven shell swapping live ↔ upcoming. Zoom launch via url_launcher guarded per-platform (iOS StoreRedirect, Android canLaunchUrl → Play Store fallback). Hero header identical to notes/videos, inner sheet rounds to r28.

07 · Tests & MCQ Bank

2-tab test catalog ("MCQ Bank" / "Custom Module") → select test list → instructions gate → exam screen with question palette, timer, MobX-observed options. test_category.dart, select_test_list.dart, test_instruction_screen.dart, new_exam_component/exam_screen.dart.

Mock Tests (masterTest)

AllTestCategoryScreen → ChooseTestScreen → SectionListScreen → SectionExamScreen → MasterTestReportScreen → LeaderboardScreen → StrengthWeaknessGraph. Dual tabs (NEET SS / INISS-ET), section-wise timing, predictive rank bands and per-topic radial graphs.

Quiz of the Day

Four daily states: countdown-until-live, live-and-ready ("Let's Go"), live-and-completed (stats + View Solutions), no-quiz-scheduled. QuizScreen → CustomStartQuizBottomSheet → QuizTestExamScreen → QuizSolutionReport.

Custom Tests

5-step learner-authored module builder: CustomTestLists → SelectCustomTestsCategory → select_custom_test_chapter → select_custom_test_topic → custom_test_select_test (optional) → CustomConfiguration → custom_preview → custom_test_exam_screen. Selected IDs stream forward through selectedCategoryItems / selectedChapterItems / selectedTopicItems / selectedExamItems.

Reports & Rankings

ReportMainScreen (regular exam) + RankListScreen (mock exam podium + leaderboard) + SolutionReport reviewer + TrendAnalysis line graphs. Uses ReportByCategoryModel, MeritListModel, forked AnimatedCircularChart.

Subscriptions (legacy flow)

Older subscription module: SubscriptionScreen (My Plan & Orders) + SubscriptionList (tabbed NEET SS / INISS-ET) + NeetGroupSubscriptionList/IniGroupSubscriptionList + SubscriptionDetailScreen + payment success/failed. Uses SubscriptionStore.

New Subscription Flow

NewSubscription → ExamGoalDialog → NewCustomSubscriptionPlan → SelectSubscriptionPlan → NewCheckoutPlan (with pincode gate) → SelectAddress → NewAddAddress → SelectDeliveryType → NewSelectOffersPlan → Razorpay → PaymentSuccessScreen. Stores: SubscriptionPlanStore, NewSubscriptionStore, OrderedBookStore.

Upgrade Plans

SelectUpgradePlanScreen — same visual shell as SelectSubscriptionPlan but fetches from UpgradePlanStore gated by two mutually exclusive flags sameValidity / isDiffValidity.

Orders & Tracking

TrackOrderScreen — product card + activity timeline driven by OrderStore.trackOrder(orderId, "", context) and rendered from the shipmentActivities array.

Bookmarks (legacy + new-bookmark-flow)

Two parallel flows: the legacy category → subcategory → topic → exam → question drill-down (BookMarkStore), and the newer BookMarkScreen1 dual-tab hub + custom-module builder (BookmarkNewStore). Question detail screens share the same Quill-annotated detail shell.

Progress

ProgressScreen — hero header + five _ProgressCards (Notes / MCQ Bank / Videos / Mock Exams / Learning Insights) fed by HomeStore.progressDetails. Desktop clamps content to 600 px via constraints.maxWidth > 600.

Watch History & Reset

DeleteHistoryScreen — "Reset Progress" module. Selectable HistoryType rows (Videos / MCQ Bank / Mock Exams / Notes / Custom Modules / MCQ Bank Bookmarks / Mock Exam Bookmarks) wired to DeleteHistoryProvider.toggleType and cleared via ApiService.deleteAllHistory(selectedTypes.toList()).

Testimonials & Blog

lib/modules/testimonial_and_blog — TestimonialScreen, BlogScreen, BlogDetailsScreen with CustomTestimonialDescriptionWidget. All three wired through HomeStore: onGetTestimonialListApiCall(), onGetBlogsListApiCall(), onGetBlogDetailsApiCall(blogId). Models preserve @JsonKey(name:'_id') sId, @JsonKey(name:'Image') image, @JsonKey(name:'blogCategory_id') blogCategoryId. Blog list uses CustomDynamicHeightGridView (3 cols) on Windows, ListView.builder elsewhere. Blog details clamps cover image to 350 px on desktop, 200 px on mobile.

Review Queue

lib/modules/review/review_queue_screen.dart — ReviewQueueScreen({super.key}) with SM-2 spaced-repetition backend. Cards auto-enqueued by server on wrong answers + manual flags. Students rate each card (again/hard/good/easy); server updates interval_days & ease_factor. Client calls ApiService().getReviewNext(limit:20) on init and ApiService().submitReviewAnswer(itemId, rating, timeSpentMs) on each rating. Time elapsed tracked via _cardStartAt timestamp captured at render.
