⚙ Engine capabilities — what each UI surface maps to
💾

Answer autosave

Every change, debounced 600ms

POST /exam/{id}/autosave with answers[]. Offline buffer up to 120s, flushes on reconnect.

⚑

Flag / skip / bookmark

Per-question metadata

Kept inside the same autosave payload. The palette sheet is purely a client derivation of answers[].

🔒

Section-lock

Immutable transitions

POST /exam/{id}/lock-section — server rejects writes for locked section_id from then on.

📝

Scratchpad

Autosaved to attempt

One free-text field added to Attempt. Survives refresh and device switch. No new route.

🧠

SM-2 self-rating

Practice mode only

Ratings feed the same spaced-repetition queue used in Review. Single algorithm across modes.

📡

Mock lobby · rivals

Live WS channel

Rank and live-rival counts reuse the socket we already run for live classes. Just a new event type.

⌨

Keyboard-first laptop

Zero extra backend

Bindings live in the client. Shown in chrome so students learn them during the session itself.

🖩

Calc & formulas

Client-only

No round-trip. Formula list is JSON-driven so staff add items without shipping a new release.

✨ What this changes for the student

Zero mystery

Timers, locks and rules are visible the whole way through.

One place for tools

Calc + pad + formulas share a floating sheet — no screen-switch context loss.

Flag & return without drift

Sticky palette chip always a tap away. Filter by flagged / skipped instantly.

Practice ≠ exam

Instant-feedback mode is visually distinct and feeds the same SR queue as review.

Mock day feels premium

Lobby, rivals live count, readiness checklist, door lock — theatre + trust.

Laptop users 2× faster

Shortcuts always on screen; ⌘↵ submit, 1-4 answer, F flag.
