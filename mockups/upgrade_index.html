How to read a screen pair
What "Today" means
A faithful approximation of the current Flutter UI based on the actual lib/modules/* files — same hierarchy, same widgets (CustomButton, SearchFiled, ListView, CarouselSlider), same spacing. Issues visible in code (blank-screen loading, hidden filters, crowded carousels, silent errors) are reflected honestly.
What "Upgraded" means
Same widget tree, same MobX store, same route, same API call — just with the new design-system tokens, tighter spacing, the absorbed mockup features where relevant, and skeleton/empty/error states filled in.
The "What changes" box under each pair
Lists the concrete deltas: new tokens applied, which mockup cluster landed, which feature flag gates the new bit (if any), which existing widget file is the target.
