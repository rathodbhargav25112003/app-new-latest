🧩 API sketches — every feature, one endpoint contract away
📊

Readiness score + forecast

Dashboard hero · forecast page

GET/api/analytics/readiness
→ { score, confidence, projected_rank, projected_percentile, drivers[], simulations[] }

Derived server-side from mock scores, study hours, topic accuracy, calendar adherence.

🏁

Percentile + cohort

Where-you-stand strip

GET/api/analytics/percentile?scope=all|topic|mock
→ { user_pct, cohort_size, bucket_counts[101], anchors: {p25,p50,p75,p90} }
🤝

Peer comparison

Anonymous cohort + nearest

GET/api/analytics/peers?scope=cohort|top10|friends
→ { metrics: [{ key, you, peer_avg, delta }], nearest: [{ alias, rank, acc, hours }] }
📈

Mock rank history

SVG trend + mock list

GET/api/analytics/mocks?limit=12
→ [{ mock_id, title, date, score, max, rank, cohort, pb }]
⏱

Time-per-question

Histogram + fast/slow matrix

GET/api/analytics/tpq?attempt_id=...
→ { median, p05, p95, buckets[], pace_acc: {fr, sr, sw, fw}, fast_wrongs[] }
📅

Study calendar

Planner blocks + completion

GET/api/planner/month?from=2026-04-01
POST/api/planner/block { kind, title, duration, start_at }
PATCH/api/planner/block/:id/complete
🤖

AI study plan

Weekly rebuild · reasoning

POST/api/planner/ai-generate { target_exam, weeks, hours_per_week }
→ { weeks: [{ w, focus:[{topic, goal, materials, hours}] }], reasoning }

Runs Sunday via cron · stores latest plan as the current calendar layer.

🎴

Flashcard decks

SM-2 scheduling

GET/api/flashcards/due → [{ id, front, back, deck, interval }]
POST/api/flashcards/:id/rate { rating:0..3 } → { next_due }
POST/api/flashcards/auto { source:"mock_wrong"|"topic" }
🕸

Concept map

Mastery-coloured graph

GET/api/analytics/concept-map?topic=shock
→ { nodes: [{ id, label, mastery }], edges: [{ a, b, strength }] }

Mastery = weighted accuracy + recency · recomputed whenever an attempt is scored.

🧪

What-if simulator

Drives the readiness page

POST/api/analytics/simulate
body { changes: [{ key, delta }] } → { projected_rank, delta }
⚡

AI insight feed

Dashboard nudge card

GET/api/ai/insights?n=3
→ [{ kind, title, body, cta_label, cta_action, urgency }]

Refreshed at login & every 6 h. Action buttons dispatch to planner / practice / flashcards directly.

🔔

Smart nudges

Push & in-app

GET/api/notifications/schedule
PATCH/api/notifications/prefs { quiet_hours, kinds[] }
✨ Why this moves the needle for students

✓Clarity of standing

Students see their rank, percentile & projected rank on one screen, every day.

✓Actionable, not pretty

Every metric pairs with an AI-suggested next action — one tap to practice, drill, flashcard.

✓Plan rebuilds itself

AI re-plans weekly based on actual performance — not what the student last committed to.

✓Peer motivation, not shame

Anonymous cohort comparison, nearest-rival deltas, gap-closing advice — never public leaderboards.

✓Simulate before you spend

The what-if panel answers "is more study worth it?" with a concrete rank projection.

✓Zero-cost parallelism

Every feature maps to an isolated endpoint. Backend and UI teams can ship independently.
