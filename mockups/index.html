UX principles applied across the redesign
01 · Clarity

One primary action per screen

Ambiguity gone. Every screen has one unmistakable next step, one accent button, and a clear hierarchy.

 02 · Continuity

Resume-first home

The thing the student was doing last is the first thing they see. No more "where was I?".

 03 · Feedback

Micro-interactions & haptics

Taps feel alive. Ratings, toggles and loading states all have real transitions, not cuts.

 04 · Typography

SF-style tightened type

Negative tracking on display sizes, optical sizing, feature flags for numerals. Reads like iOS.

 05 · Surfaces

Soft depth, not heavy shadows

Three shadow tiers, glass/blur on floating bars, generous radii. Dark mode uses true blacks.

 06 · Accessibility

44pt targets, WCAG AA contrast

Rating buttons, FABs, list rows — all tap-friendly on every device. Dark mode passes AA too.

Responsive strategy
Phone

Single-column, bottom tab bar

Glass bottom tab bar floats 10pt off screen edge. FAB cluster stacks vertically above the tab bar.

 iPad mini (744)

Expanded single pane

Wider content, same navigation. Cards grow horizontally; Resume banner shows 4 cards instead of 2.5.

 iPad Pro / tablet

Split-view master-detail

Sidebar navigation replaces bottom tabs. Video player + notes side-by-side. Progress is a 2-column dashboard.

 Laptop / macOS

Three-column app shell

Rail (modules) · list (content) · detail (player/reader). Keyboard shortcuts surfaced in tooltips.
